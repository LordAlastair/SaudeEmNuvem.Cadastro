As classes base fornecidas pelo SeedWord evita que a equipe de desenvolvimento tenha código redundante nas classes de objetos de cada domínio.
A pasta para esses tipos de classes é chamada SeedWork ela não é algo como uma Framework.
Essa pasta é chamada de SeedWork, pois contém apenas um pequeno subconjunto de classes reutilizáveis que realmente não pode ser considerada uma estrutura.
Seedwork é um termo introduzido por difusões de Michael e popularizada por Martin Fowler muitas vezes como sharedKernel:
https://martinfowler.com/bliki/Seedwork.html
